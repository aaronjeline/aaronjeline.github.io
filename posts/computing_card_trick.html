The problem statement

Formalizing the problem:

For a game of Texas Hold ’Em with with n players, does there exist a deck of cards ( d ), such that, No matter what position the deck is cut at (0-52), player 0 wins the next hand?

(Magicians believe the answer is ‘yes’)

Setup: Representing Decks

My initial idea was to try to find deck through random search. Given that there are a lot of decks of cards ( 52! ), I figured performance was going to matter and some parallelism would probably be involved. Thus, I picked Rust as my implementation language.

Given that there are 52 values for a given card, a card is represented a single u8 : 

pub struct Card(pub u8);

A deck of cards is just an ordered sequence of cards, so it’s a 

pub struct Deck(pub Vec<Card>);

impl Deck {
  /// Constructor for new decks
  pub fn new_deck_order() -> Self { ... }
  /// Our operations on decks
  pub fn cut(self, pos : usize) -> Self {}
  pub fn draw(&mut self) -> Card {}
  pub fn shuffle(self) -> Card {}
  /// Swap two cards at positions x and y
  pub fn swap(&mut self, x : usize, y : usize)
}

With a simple proptest that cutting the deck twice should be a no-op 

fn cut_twice_roundtrip(cut_pos in 0usize..52) {
  let deck = Deck::new_deck_order();
  let d1 = deck.clone().cut(cut_pos);
  let d2 = d1.cut(52 - cut_pos);
  assert_eq!(deck, d2);
}

Try #2: Hill Climbing

Okay, so that didn’t work. Well let’s go simpler. We will try standard hill climbing with random restarts. Fortunatly, we’re in a discrete space. So we can know with confidence if we are in a local maxima. The algorithm becomes this: 1. Pick a random deck 2. Check the number of wins: if it’s 52: we’re done! 3. Try every pair of swaps, keeping the one that wins you the most games 4. If no swap increases your score -> goto 1 Reuslts: This is a very hilly landscape. There are lot’s of decks that win 50 or 51 games, and many of them have no swaps that increase wins.
